Model Trainer — hub

Pipeline versions

Each version is a snapshot of how data is fed + scored. A BREAKING version makes prior runs' scores incomparable; runs are tagged with the version they ran under.

Datasets

A named set of dataset settings (asset, walk-forward window, fidelity stack — e.g. 1h vs 1h+1d) — which data a model trains and tests on, not the model itself. Run a model against several to see how it holds up across datasets.

Environments

A named set of environment settings (fees, take-profit / stop-loss) — the market a model trades in, not the model itself. Run a model against several to see how it holds up across regimes.

Hypotheses

Claims that runs prove or disprove — each verdict (untested / proven / disproved) auto-derives from the runs matching its spec and re-checks as new runs land.

Papers / Library

Approaches to prove out or falsify — a container of the testable hypotheses it makes (extract them from the link, add your own, or link existing ones). Its verdict rolls up from those hypotheses; it "holds up" only when one beats buy-and-hold out-of-sample, net of fees.

Models

The model architectures this project can train — each linked to the papers that introduce or improve it, the runs that train it, and the hypotheses that test it. Status (proposed / implemented / failing) rolls up from its runs. Discuss a model with the AI to implement, improve, or fix it.
